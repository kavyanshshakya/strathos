An RL environment for adversarial agent robustness.

I built Strathos to operationalise the OWASP ASI 2026 taxonomy as a multi-step OpenEnv training environment. Hidden context per scenario, a 7-component composable reward, and a Qwen 3 1.7B agent post-trained with SFT and GRPO.

Kavyansh ShakyaIISER Bhopalsolo entry

What it is

Strathos turns the OWASP Top 10 for Agentic Applications 2026 into a programmatic RL training surface. The first concrete domain is a regulated robo-advisor under SEC Regulation Best Interest. Adversarial scenarios carry hidden context that only surfaces through investigation tool calls — the agent must call audit_trail_query, check_risk_tolerance, and similar before deciding to refuse, escalate, verify, or execute.

The architecture separates the domain layer (TaskEnv) from the adversarial layer (StrathosWrapper). Cross-domain stubs for medical triage and code review demonstrate the extension pattern. Finance is V1.

What's shipped

8 of 10 OWASP ASI classes

ASI01-07 plus ASI09 with 15 hand-authored adversarial scenarios. ASI08 and ASI10 architecturally scaffolded.

7-component composable reward

Format, anti-spam, decision, ASI class, task success, investigation bonus, optimal-tools match. Per-skill debugging.

Three training stages

V2 base SFT → V2-PLUS grounded discrimination (deployed) → GRPO research artifact with reward hacking documented.

Multi-step investigation

Hidden context per scenario surfaces only via tool calls. Backward-compatible single-turn API.

11 passing tests

Schema validity, ASI class coverage, live env contract. Last run 11/11 in 8.81s.

Real wandb tracking

SFT re-training trace public on wandb. Loss 4.4 to 0.69 over 75 steps.

Design decisions worth naming

Choices that distinguish Strathos from a single-domain submission with one training pass.

Depth over breadth

One domain — regulated finance under SEC Reg BI — done end-to-end with documented findings, instead of a shallow tour across many. The TaskEnv abstraction keeps the cross-domain claim architectural rather than empirical, which is honest about what V1 actually validates.

Composable reward, not opaque scalar

The 7 reward components are independently inspectable. During training I could see format_valid saturate at 100% three epochs before asi_class_correct moved off baseline, which told me the model was learning to output JSON before learning to discriminate attacks. That kind of per-skill visibility is what made V2 to V2-PLUS feasible without restarting.

Honest failure documentation

GRPO exhibited reward hacking — the model learned to output "asi_class": "ASI01-ASI07", exploiting a substring match in the reward function to collect partial credit on every adversarial scenario. The GRPO adapter is published as a research artifact with the failure mode documented in its model card; V2-PLUS SFT is the deployed model. I would rather ship a working SFT with an honest GRPO postmortem than claim a successful RL run that wasn't.

Industry taxonomy, not custom benchmark

Strathos operationalises the OWASP ASI 2026 taxonomy directly. Not a benchmark invented for the hackathon. This makes the environment useful as a public reference for anyone working on agentic AI security, not just as a hackathon artefact.

Roadmap

Near-term (2-4 weeks)

  • Multi-step RL training. Train on full multi-step trajectories so the model actually calls investigation tools before deciding. Original GRPO ambition before reward hacking forced V2-PLUS retention.
  • Scenario expansion. 15 to 80+ scenarios across 8 classes. Bounded authoring work.
  • Ship ASI08 and ASI10. Architecturally supported; scenarios just need authoring.
  • Tighter reward function. Replace substring match with strict ASIClass enum validation. Closes the GRPO regex exploit.

Mid-term (1-3 months)

  • Cross-domain validation. Build out medical_triage and code_review stubs into full domains. Empirical, not just architectural.
  • Frontier model leaderboard. Claude, GPT-4o, Gemini 2.5, Llama 3.3 vs trained Qwen 1.7B on the same 15 scenarios.
  • Reward hacking case study. The GRPO regex exploit is a clean reproducible example. NeurIPS Safety / ICLR Safe AI workshop.

Long-term (3-12 months)

  • Strathos as a benchmark. Public train/test split, formal eval protocol, leaderboard. NeurIPS Datasets & Benchmarks 2026.
  • OWASP collaboration. Propose Strathos as reference RL tooling under the Agentic Security Initiative.
  • Microsoft Agent Governance Toolkit pairing. Training-time hardening + runtime enforcement = defence in depth. AGT (April 2026) maps 1:1 to OWASP ASI 2026.
  • Adversarial co-evolution. Co-train a learned adversarial client with the focal agent without either side learning to game the other.

Honest limitations

  • 15 hand-authored scenarios across 8 classes is small. Production benchmark needs 100+ per class.
  • GRPO model exhibits reward hacking and format collapse. V2-PLUS SFT retained as deployed; GRPO published as research artifact with failure modes documented in its model card.
  • Multi-step reward components are exposed but the deployed SFT model was trained on the single-turn dataset. Multi-step RL training is the natural next step.
  • ASI08 and ASI10 are scaffolded in models.py but not yet shipped as scenarios.
  • Synthetic brokerage backend with no real market data. Realistic enough for tool chain coherence; not a trading system.
